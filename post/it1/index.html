1.C语言文件读写函数基本知识与易错点总结

1.1 二进制读写函数fread()以及fwrite()
type 说明
r或rb 为读而打开，文件必须已经存在，不存在则报错
w或wb 把文件截断至0长（文件存在，先把文件清空，再写），或为写而创建（文件不存在会创建）
a或ab 追加；为在文件尾写而打开（文件已经存在，则在原有内容后面追加），或为写而创建（文件不存在，会先创建）
r+或r+b或rb+ 为读和写而打开，若欲操作的文件不存在，则打开失败。 成功打开文件时，文件指针位于文件开头。打开文件后，不会清空文件内原有内容。无论是读取内容还是写入内容，都可在文件中任意位置进行，且进行写入操作时，会覆盖原有位置的内容。
w+或w+b或wb+ 把文件截断至0长（文件存在，先把文件清空，再写），或为读和写而打开，文件不存在会先创建
a+或a+b或ab+ 在文件尾读和写而打开或创建 
**附表格加强记忆：**
打开模式 简述 若欲操作的 文件不存在 成功打开文件后 文件指针位置 是否清空 原有内容 读取位置 写入位置 注
r 只读 打开失败 开头 否 任意位置读取 不可写入 -
w 只写 新建 开头 是 不可读取 任意位置写入 写入时会覆盖原有位置内容
a 新建 结尾 否 不可读取 只能尾部写入 -
r+ 读写 打开失败 开头 否 任意位置读取 任意位置写入 写入时会覆盖原有位置内容
w+ 新建 开头 是 任意位置读取 任意位置写入 写入时会覆盖原有位置内容
a+ 新建 结尾 否 任意位置读取 只能尾部写入 -

1.3 字符输入输出函数fgetc()与fputc()
  • 在fgetc函数调用中，读取的文件必须是以读或读写方式打开的。
  • 读取字符的结果也可以不向字符变量赋值。例如：fgetc(fp); 但是读出的字符不能保存。
  • 在文件内部有一个位置指针。用来指向文件的当前读写字节。在文件打开时，该指针总是指向文件的第一个字节。使用fgetc 函数后，该位置指针将向后移动一个字节。 因此可连续多次使用fgetc函数，读取多个字符。应注意文件指针和文件内部的位置指针不是一回事。文件指针是指向整个文件的，须在程序中定义说明，只要不重新赋值，文件指针的值是不变的。文件内部的位置指针用以指示文件内部的当前读写位置，每读写一次，该指针均向后移动，它不需在程序中定义说明，而是由系统自动设置的。

1.4 字符串读写函数fgets()与fputs()

用法同fgetc和fputc

fgets()遇到回车才停止，遇到空格不停止

此外，fgets()虽然遇到回车停止，但是它会把换行符也读进去，然后在串尾加上\n，所以它后面就不用跟上fgetc（）来吃掉换行符。

如果一个文件的当前位置的文本如下

Love, I Have

Since you can do it.

如果用fgets(str1,6,file1);去读取

则执行后str1 = “Love,” ，读取了6-1=5个字符

这个时候再执行fgets(str1,20,file1)则执行后str1 = “ I Have\n”，注意

而如果

fgets(str1,23,file1);

则执行str1=“Love ,I Have”，读取了一行(包括行尾的’\n’,并自动加上字符串结束符’\0’)，当前文件位置移至下一行，虽然23大于当前行上字符总和，可是不会继续到下一行。而下一次调用fgets()继续读取的时候是从下一行开始读。

从 handle 指向的文件中读取一行并返回长度最多为 length - 1 字节的字符串。碰到换行符(包括在返回值中)、EOF 或者已经读取了 length - 1 字节后停止(看先碰到那一种情况)。如果没有指定 length，则默认为 1K，或者说 1024 字节。

出错时返回 FALSE。

1.5变量类型与进制转换sscanf()与sprintf()

实际问题中常常遇到如何进行字符串与数的相互转换，这个时候，这两个函数就派上用场了

实例，用C语言模仿python的readline函数

//line1 和 line2 指定读取文件中的line1-line2行所有的行并存到二维数组中
void readline(char(*buffer)[300],char filenametxt[],int line1,int line2){
	FILE *fd;
	int i,j;
	fd=fopen(filenametxt,"r");
	rewind(fd);
	for(i=0;i<line1;i++) {fscanf(fd,"%s",buffer[0]);fgetc(fd);}
	for(j=0,i=line1;i<=line2;i++,j++) {fscanf(fd,"%s",buffer[j]);fgetc(fd);}
	fclose(fd);
	return ;
}

点评一下，这里我为什么不用fgets()，因为它会把空格也读进去，而在打开文件时，文件名字符串末尾包含空格根本无法打开。即fopen("1.dat ","wb")会报错
